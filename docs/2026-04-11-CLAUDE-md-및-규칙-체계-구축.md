## 작업 내용
- `CLAUDE.md` 생성: 프로젝트 개요, 기술 스택, 구조, 아키텍처, 코드 컨벤션 정리
- `docs/` 디렉토리 생성 및 대화 기록 문서화 체계 도입
- `.claude/rules/conversation-docs.md` 생성: 대화 기록 지침을 별도 규칙 파일로 분리

## 주요 결정 사항
- CLAUDE.md는 프로젝트 개요/컨벤션만 유지하고, 행동 지침은 `.claude/rules/`로 분리
  - 이유: CLAUDE.md가 커지면 토큰 낭비. rules 파일은 항상 자동 로드되므로 기능 동일
  - `docs/` 하위에 넣는 방안은 해당 디렉토리 작업 시에만 로드되어 부적합
- 대화 기록은 `docs/YYYY-MM-DD-제목.md` 형식, 매번 사용자 확인 후 생성
