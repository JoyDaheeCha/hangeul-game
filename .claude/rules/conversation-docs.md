# 대화 기록 문서화

대화가 끝나갈 때 (작업 완료, 커밋/푸시 후 등) 사용자에게 이번 대화 내용을 `docs/`에 정리할지 물어본다.

사용자가 동의하면:
1. `docs/YYYY-MM-DD-간단한제목.md` 파일을 생성
2. 아래 템플릿을 따른다
3. 커밋 메시지: `docs: YYYY-MM-DD 대화 기록 추가 — 제목`

## 문서 템플릿
```markdown
# 제목 (작업 요약)
> YYYY-MM-DD

## 작업 내용
- 무엇을 했는지 (변경된 파일, 추가된 기능, 수정된 버그 등)

## 주요 결정 사항
- 왜 이런 방식을 선택했는지, 대안은 무엇이었는지

## 다음 할 일 (있다면)
- 후속 작업이나 남은 이슈
```
